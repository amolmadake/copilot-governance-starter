# Copilot Governance Starter

A **2-stage AI governance pipeline** that auto-generates GitHub Copilot configuration for any repository — instructions, prompts, skills, and agents — all derived from your actual codebase architecture.

## Why

Setting up Copilot agent mode properly requires many files: `copilot-instructions.md`, instruction files, prompt files, skill files, agent definitions. Writing them manually is tedious and they drift out of sync with the codebase.

This pipeline **scans your repo, detects the architecture, and generates everything automatically**.

## How It Works

```
┌─────────────────────────────────────────────────┐
│  Stage 1: generate-architecture.prompt.md       │
│  (manual file — the only thing you copy)        │
│                                                 │
│  Scans repo → detects stack, modules, deps      │
│  Outputs: overview.md, modules.md,              │
│           generate-copilot.prompt.md             │
├─────────────────────── ▼ ───────────────────────┤
│        Developer reviews & edits if needed       │
├─────────────────────── ▼ ───────────────────────┤
│  Stage 2: generate-copilot.prompt.md            │
│  (auto-generated by Stage 1)                    │
│                                                 │
│  Reads arch docs → derives governance rules     │
│  Outputs: 18 Copilot config files               │
└─────────────────────────────────────────────────┘
```

There is a **manual review gap** between stages so developers can verify and correct the architecture docs before they flow into all config files.

## Quick Start

### Use in this repo

```bash
# Stage 1 — open in VS Code, run with Copilot agent mode
docs/ai-pipeline/generate-architecture.prompt.md

# Review the 3 output files, edit if needed

# Stage 2 — open in VS Code, run with Copilot agent mode
docs/ai-pipeline/generate-copilot.prompt.md

# Review the 18 output files — done
```

### Use in any other repo

1. Copy **one file** into your repo:
   ```
   docs/ai-pipeline/generate-architecture.prompt.md
   ```
2. Open it in VS Code and run with Copilot (agent mode)
3. Review the Stage 1 outputs, edit if needed
4. Run the auto-generated `generate-copilot.prompt.md`
5. Review Stage 2 outputs — done

See [docs/ai-pipeline/README.md](docs/ai-pipeline/README.md) for the full setup guide.

## What Gets Generated (22 files)

| Category | Count | Files | Type |
|----------|-------|-------|------|
| Architecture docs | 2 | `docs/architecture/overview.md`, `modules.md` | Repo-specific |
| Stage 2 prompt | 1 | `docs/ai-pipeline/generate-copilot.prompt.md` | Repo-specific |
| Governance context | 1 | `docs/architecture/context.md` | Repo-specific |
| Agent index | 1 | `AGENTS.md` | Repo-specific |
| Copilot config | 1 | `.github/copilot-instructions.md` | Repo-specific |
| Instructions | 2 | `.github/instructions/*.instructions.md` | Repo-specific |
| Prompts | 5 | `.github/prompts/*.prompt.md` | Generic |
| Skills | 3 | `.github/skills/*/SKILL.md` | Generic |
| Agents | 5 | `.github/agents/*.agent.md` | Generic |

**Repo-specific** files contain your actual stack, modules, and conventions.
**Generic** files work on any repo — they reference `.github/instructions/` for framework details.

## Supported Stacks

The bootstrap prompt detects any stack automatically. Tested patterns:

- **NestJS** — Resolver → Service layering (this repo)
- **Spring Boot** — Controller → Service → Repository layering
- **Express / Fastify** — Route → Handler → Service layering
- **.NET** — Controller → Service → Data Access layering
- **Django** — View → Serializer → Model layering

The pipeline reads `package.json`, `pom.xml`, `go.mod`, `requirements.txt`, `*.csproj`, etc. to detect the stack and derive appropriate conventions.

## Updating

When your architecture changes:

1. Re-run Stage 1 → updates architecture docs
2. Review edits
3. Re-run Stage 2 → updates all 18 config files

All files support **upsert** — create if missing, update if they exist.
