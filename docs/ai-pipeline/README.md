# AI Governance Pipeline — Setup Guide

Use this guide to set up the AI governance pipeline in any repository.

## Prerequisites

- VS Code with GitHub Copilot (agent mode)
- Any codebase (NestJS, Spring Boot, Express, Django, .NET, Go, etc.)

## How It Works

The pipeline runs in **2 stages**, each driven by a prompt file:

```
Stage 1: generate-architecture.prompt.md  →  scans repo  →  produces architecture docs + Stage 2 prompt
Stage 2: generate-copilot.prompt.md       →  reads arch docs  →  produces 18 governance files
```

Stage 1 is the only manual file. Everything else is auto-generated.

## Step-by-Step Setup

### 1. Copy the bootstrap prompt

Copy `docs/ai-pipeline/generate-architecture.prompt.md` into your target repository at the same path:

```
your-repo/
└── docs/
    └── ai-pipeline/
        └── generate-architecture.prompt.md    ← copy this file
```

This is the **only file you need to copy**. It is generic and works on any stack.

### 2. Run Stage 1

Open `generate-architecture.prompt.md` in VS Code and run it with Copilot (agent mode).

Stage 1 will:

- Scan your repository (package.json, pom.xml, go.mod, etc.)
- Detect your stack, architecture, modules, and dependencies
- Generate 3 files:

```
docs/architecture/overview.md          ← tech stack, structure, data flow
docs/architecture/modules.md           ← module map, dependency rules
docs/ai-pipeline/generate-copilot.prompt.md  ← Stage 2 prompt (auto-generated)
```

### 3. Review Stage 1 output

**Stop and review** the 3 generated files. These are your architecture source of truth — everything in Stage 2 derives from them.

Edit if needed:

- Fix any incorrect version numbers in `overview.md`
- Adjust module boundaries in `modules.md`
- The Stage 2 prompt (`generate-copilot.prompt.md`) can also be edited if you want to customize what it generates

### 4. Run Stage 2

Open `generate-copilot.prompt.md` and run it with Copilot (agent mode).

Stage 2 will read `overview.md` and `modules.md`, then generate **18 files**:

| Category           | Files                                               | Type          |
| ------------------ | --------------------------------------------------- | ------------- |
| Governance context | `docs/architecture/context.md`                      | Repo-specific |
| Agent index        | `AGENTS.md`                                         | Repo-specific |
| Copilot config     | `.github/copilot-instructions.md`                   | Repo-specific |
| Instructions       | `.github/instructions/architecture.instructions.md` | Repo-specific |
| Instructions       | `.github/instructions/testing.instructions.md`      | Repo-specific |
| Prompts            | `.github/prompts/` (5 files)                        | Generic       |
| Skills             | `.github/skills/` (3 files)                         | Generic       |
| Agents             | `.github/agents/` (5 files)                         | Generic       |

### 5. Review Stage 2 output

**Stop and review** the 18 generated files.

- Files 1–5 are **repo-specific** — derived from your architecture. Check for accuracy.
- Files 6–18 are **generic** — they reference `.github/instructions/` for framework details. Usually fine as-is.

## Updating

When your architecture changes (new modules, new libraries, etc.):

1. Re-run Stage 1 (`generate-architecture.prompt.md`) — it will update `overview.md` and `modules.md`
2. Review the updated architecture docs
3. Re-run Stage 2 (`generate-copilot.prompt.md`) — it will update all 18 files

All files support **upsert** — they create if missing, update if they exist.
